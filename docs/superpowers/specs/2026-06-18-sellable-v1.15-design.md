# Sellable v1.15 — Trust/Polish Remediation + Cost Widget — Design

**Goal:** Make DeepotusVideoGen v1.15 a credible *paid* one-time download for **crypto/Web3 creators**, sold on the Deepotus site (BYO API keys), with margins funding the rippled wallet. Add a **Cost Widget** (per-API usage + remaining + monetary value, and a preview budget before any operation) as a headline selling point.

**Audience:** crypto/Web3 creators (memecoin communities, project marketers, KOLs). Semi-technical; BYO-keys acceptable.

**Distribution model:** Phase 1 = one-time download, BYO keys (current architecture). Phase 2 (later, demand-driven) = credit-tier subscriptions.

**Hard constraints:**
- Frontend has **no source** — only the minified bundle `frontend/dist/assets/index-BEOJX8L5.js`. All UI changes are surgical Python-script patches (unique-anchor replace) validated with `node --check` + browser verify. See [[frontend-compiled-only]].
- Runtime is **Python 3.14** (PEP 649 deferred annotations).
- Local app, single-user, localhost-bound. BYO keys live in `%LOCALAPPDATA%\DeepotusVideoGenData\.env`.
- Every change re-bundled into the v1.15 installer (Inno Setup) + portable export.

## Part 1 — Path A: Trust & Polish Remediation

### 1.1 Trust & security (launch blocker)
- **Re-enable TLS verification by default** on all outbound `httpx` calls (fal_service, heygen_service, openai_llm, gemini_llm, marketing, summarizer, news_service, article_scraper, routes image fetch). Replace bare `verify=False` with a module-level `VERIFY = os.environ.get("DEEPOTUS_INSECURE_SSL","")!="1"` and pass `verify=VERIFY`. Keep the truststore/OS-cert injection in `main.py` (that *helps* verification) but **remove the unconditional `ssl.create_default_context` strict-flag monkey-patch** — make it apply only when `DEEPOTUS_INSECURE_SSL=1`.
- **Endpoint hardening** (`routes.py`): apply the existing `_require_localhost` guard to state-changing / credit-spending endpoints: `/generate`, `/generate/batch`, `/generate/heygen`, `/generate/composition`, `/images/generate`, `/images/import-url`, `/images/upload`, `/videos/upload`, `/marketing/plan*`, `/schedule*` (POST/PATCH/DELETE/fire), `/channels/test`, `/news/refresh`, `/news/sources*`.
- **Sanitize `/images/upload` filename** (path traversal): `Path(file.filename).name` only; reject separators.
- **Restrict `/images/import-url`** (SSRF): require `http(s)` scheme, enforce size cap, no redirects to non-http schemes.

### 1.2 Honest "AI" claims
- The deterministic copy paths (`prompt_engine.build*`, `generate_news_script` tone) currently produce persona scaffolds, not LLM output. **Route them through the existing multi-provider LLM** (reuse `summarizer`/`marketing` provider plumbing) when a key is configured; **fall back to the deterministic scaffold** otherwise. The "AI" label then matches reality. Align guide + sales-page wording.

### 1.3 Polish — remove all "mockup smell"
- **Library**: wire **search** + **sort** (filter/sort the real assets list; mirror the News fix).
- **Studio**: replace the hardcoded "Output / Estimated" panel (`tweet_2026-05-20_oracle`, `00:17.6`, `$0.43`, fixed providers) with **real graph state** (editable graph name; duration/cost shown as "estimated at run" or computed via the Cost Widget estimator from Part 2).
- **Quick**: wire or remove the stub `Voice (HeyGen comp)` toggle and the stub search box.
- **Final stub sweep** of the bundle for any remaining `onChange:()=>{}` / missing `onClick` on user-facing controls.

### 1.4 Correctness niceties
- `prompt_engine.py`: add `VoiceMode` to the `app.models.schemas` import (kills latent NameError under eager-annotation runtimes / `get_type_hints`).
- `composition_service.py`: sequential compositions are silently muted (the with-audio `filter_complex` branch is dead). Either wire it or clearly label sequential as silent in the UI/docs.

### 1.5 Distribution & onboarding
- **Onboarding/docs**: add a "Get your API keys in ~5 min" quickstart and a **key → capability matrix** (fal.ai required; HeyGen/ElevenLabs/LLM/Telegram/X optional) to the guide + first-run wizard.
- **Code-signing** the installer is strongly recommended for a paid download (removes SmartScreen friction). **Business action** (needs an OV/EV cert) — documented, not built here.

### Non-goals (Phase 2)
YouTube/Instagram auto-publishers; credit-tier subscriptions + license/activation server; hosted multi-tenant SaaS.

## Part 2 — Cost Widget (new feature)

**Value:** before launching a campaign, a scheduled plan, or even a single edit, the creator sees an itemized **preview budget**; and a persistent widget shows **per-API usage, remaining balance (where available), and monetary value** in real time.

### 2.1 Backend

**`services/pricing.py`** — configurable pricing model.
- Loads defaults, overridable via `pricing.json` in the data dir (editable in Settings).
- Default unit costs (directional; user-editable):
  - fal.ai FLUX image: `$0.003` / image
  - fal.ai Seedance: `$0.04` / second of video
  - HeyGen avatar: `6` credits / minute; `credit_usd` configurable (default `$0.04`)
  - ElevenLabs: `$0.00024` / character (configurable)
  - LLM (per 1M tokens, in/out) per provider: anthropic-haiku, gpt-4o-mini, gemini-flash — configurable defaults
- `estimate(op: dict) -> {breakdown:[{provider, units, unit, usd}], total_usd, credits:{provider:n}}`.
  - `op` kinds: `image`, `seedance` (duration_s), `heygen` (script length→minutes est.), `composition` (sum of parts), `news_reel` (per_card_s × items), `marketing_plan` (posts → images+clips), `campaign` (list of posts/ops).

**`POST /api/cost/estimate`** — body = an op or list of ops → returns the breakdown + total. Used for the preview budget. Localhost-guarded (read-only compute, but consistent).

**`GET /api/cost/balances`** — live balances where the provider exposes them:
- HeyGen → `/v2/user/remaining_quota` (reuse `HeyGenClient.remaining_quota()`): `remaining` credits + `credit_usd` → `$`.
- ElevenLabs → `/v1/user/subscription`: `character_count`/`character_limit`.
- fal.ai, Anthropic, OpenAI, Gemini → **no balance API**: return `{available:false, mode:"pay-as-you-go"}`.

**`GET /api/cost/usage`** — cumulative estimated spend from job records. Add a nullable `cost_usd` column to `jobs` (auto-migrate, like prior columns); set it at job creation from `pricing.estimate`. Aggregate per provider + total (today / all-time).

### 2.2 Frontend (bundle patches)

- **Persistent Cost Widget** in the top bar (next to the provider pills): compact pills per active provider showing **used $ (cumulative)** and **remaining** (credits + `$`) where available; click to expand a panel with the full breakdown (per-API used / remaining / monetary) + a "pay-as-you-go" note for providers without balances. Polls `/api/cost/usage` + `/api/cost/balances` (~30s, reuse the existing health-poll cadence).
- **Preview budget hooks** — before a spend action, show the estimate:
  - **Quick** generate (Seedance/HeyGen/Composition): an "~$X" line near the Generate button (the form already shows an est-cost line — make it real via `/api/cost/estimate`).
  - **Scheduler**: "Produce" (single) and **plan/campaign** materialization → an itemized estimate in the modal before "Add to calendar" / before Produce.
  - **Studio**: "Run" → estimate from the compiled graph (also feeds 1.3 Studio panel).
- All numbers come from the backend estimator (single source of truth); the widget never hardcodes prices.

### 2.3 Settings
- A "Pricing & budget" section in Settings to edit the unit costs / credit-$ rates (writes `pricing.json`) and set an optional **monthly budget cap** (the widget warns when projected spend would exceed it).

## Verification
- Backend: unit-test `pricing.estimate` for each op kind; hit `/api/cost/estimate|usage|balances` and assert shapes; confirm TLS now verifies (a call to a known-good host succeeds; bad cert fails unless `DEEPOTUS_INSECURE_SSL=1`).
- Frontend: `node --check` after every bundle patch; in-browser verify each wired control fires its endpoint and the widget reflects real numbers (DOM/network checks, as used for News/Scheduler).
- Repackage installer; silent test-install + smoke import; confirm all changes present.

## Risks
- Minified-bundle patches are the main risk → mitigated by unique-anchor replace + `node --check` + browser verification + per-step backups.
- Pricing accuracy is directional, not billing-grade → clearly labeled "estimate"; user-editable rates.
- Re-enabling TLS could resurface the original SSL issue on locked-down networks → the `DEEPOTUS_INSECURE_SSL=1` escape hatch + truststore OS-cert use covers that.
